# Generative AI Fundamentals: What I Learned from IBM

Generative AI represents the most significant shift in AI capabilities since deep learning. Here's what I learned from IBM's Generative AI Fundamentals specialization.

## What is Generative AI?

Generative AI refers to AI systems that can create new content:
- **Text**: Articles, code, poetry, conversations
- **Images**: Art, photorealistic images, designs
- **Audio**: Music, speech, sound effects
- **Video**: Animations, deepfakes, generated footage
- **Code**: Programs, scripts, algorithms

Unlike discriminative AI (which classifies or predicts), generative AI creates.

## Large Language Models (LLMs)

### Architecture

**Transformer Architecture**
- Self-attention mechanisms
- Multi-head attention
- Positional encoding
- Feed-forward networks

**Key Models**
- **GPT (Generative Pre-trained Transformer)**: OpenAI's decoder-only model
- **BERT**: Bidirectional encoder for understanding
- **T5**: Text-to-text transfer transformer
- **LLaMA**: Meta's open-source model
- **Claude**: Anthropic's safety-focused model

### How LLMs Work

1. **Pre-training**: Learning language patterns from vast text corpora
2. **Fine-tuning**: Adapting to specific tasks
3. **Inference**: Generating responses token by token
4. **Context window**: Limited tokens the model can process

### Training Process

**Data Preparation**
- Web crawling (Common Crawl, C4)
- Book corpora
- Code repositories
- Curated datasets

**Tokenization**
- Breaking text into tokens (subwords)
- BPE (Byte Pair Encoding)
- WordPiece
- SentencePiece

**Training Objectives**
- Next token prediction
- Masked language modeling
- Instruction tuning
- RLHF (Reinforcement Learning from Human Feedback)

## Prompt Engineering

### Core Principles

**Clarity**
- Be specific about what you want
- Provide context
- Use examples (few-shot prompting)

**Structure**
- Role definition: "You are an expert..."
- Task description: "Write a..."
- Constraints: "Limit to 500 words..."
- Format: "Use bullet points..."

### Advanced Techniques

**Chain-of-Thought**
- Encourages step-by-step reasoning
- "Let's think through this..."
- Improves accuracy on complex tasks

**Few-Shot Learning**
- Provide examples in the prompt
- Pattern: Input → Output
- Helps model understand the task

**System Prompts**
- Define behavior and constraints
- Set the "personality" of the AI
- Persistent across conversation

**Temperature and Top-p**
- Temperature: Creativity vs determinism (0-2)
- Top-p (nucleus sampling): Diversity control
- Max tokens: Output length limit

### Prompt Patterns

**RTF Pattern**
- Role: "You are a technical writer"
- Task: "Explain quantum computing"
- Format: "Use analogies and examples"

**Chain-of-Thought**
```
Q: What is 23 + 47?
A: Let's break this down:
   - 20 + 40 = 60
   - 3 + 7 = 10
   - 60 + 10 = 70
   So the answer is 70.
```

**Tree of Thoughts**
- Explore multiple reasoning paths
- Evaluate each path
- Select the best solution

## Image Generation

### How It Works

**Diffusion Models**
- Start with random noise
- Iteratively denoise
- Guided by text prompt
- Stable Diffusion, DALL-E, Midjourney

**GANs (Generative Adversarial Networks)**
- Generator creates images
- Discriminator evaluates realism
- Adversarial training
- StyleGAN for faces

### Key Concepts

**Prompt Engineering for Images**
- Descriptive keywords
- Style references
- Composition control
- Negative prompts (what to avoid)

**Parameters**
- Steps: Iteration count (20-50 typical)
- CFG Scale: Prompt adherence (7-12 typical)
- Seed: Reproducibility
- Resolution: Output size

## Challenges and Limitations

### Hallucinations

**What Are They?**
- Confident but incorrect information
- Made-up facts, citations, or data
- Plausible-sounding nonsense

**Mitigation**
- Retrieval-Augmented Generation (RAG)
- Fact-checking systems
- Human-in-the-loop
- Confidence scoring

### Bias and Fairness

**Sources of Bias**
- Training data reflects societal biases
- Underrepresentation of groups
- Annotation bias

**Mitigation**
- Diverse training data
- Bias detection tools
- Fairness metrics
- Inclusive design

### Intellectual Property

**Concerns**
- Training on copyrighted material
- Generated content ownership
- Trademark infringement
- Plagiarism detection

**Approaches**
- Opt-out mechanisms
- Watermarking
- Attribution systems
- Legal frameworks

### Environmental Impact

**Energy Consumption**

**Mitigation**

- Edge deployment
