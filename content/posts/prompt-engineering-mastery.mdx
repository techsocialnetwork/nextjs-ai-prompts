# Prompt Engineering Mastery: From Basics to Advanced Techniques

Prompt engineering is the skill of effectively communicating with Large Language Models (LLMs). This guide covers everything from basic principles to advanced techniques I've learned through the IBM AI curriculum and hands-on practice.

## What is Prompt Engineering?

Prompt engineering is the practice of designing inputs (prompts) that elicit desired outputs from AI models. It's a critical skill because:

- **Model behavior is prompt-dependent**: Same model, different results based on input
- **No coding required**: Natural language interface to AI capabilities
- **Rapid iteration**: Quick testing and refinement
- **Universal application**: Works across GPT, Claude, Llama, and other LLMs

## Fundamental Principles

### 1. Clarity and Specificity

**Bad Prompt:**
```
Write about dogs.
```

**Good Prompt:**
```
Write a 300-word blog post about Golden Retrievers as family pets. 
Target audience: First-time dog owners.
Tone: Friendly and informative.
Include: Temperament, exercise needs, and grooming requirements.
```

**Why it works:**
- Specific topic (Golden Retrievers, not all dogs)
- Defined length (300 words)
- Clear audience (first-time owners)
- Specified tone (friendly, informative)
- Required elements listed

### 2. Context Setting

Provide background information the model needs:

```
Context: I'm preparing a presentation for C-suite executives about 
our Q4 marketing performance.

Task: Analyze the following metrics and identify 3 key insights:
[metrics data]

Format: Bullet points with supporting data, suitable for executive summary.
```

### 3. Role Definition

Assigning a persona improves output quality:

```
You are an experienced DevOps engineer with 10 years of AWS experience.

Review this CloudFormation template and identify:
1. Security vulnerabilities
2. Cost optimization opportunities  
3. Best practice violations

Template:
[CloudFormation YAML]
```

Common roles:
- Expert consultant in [field]
- Experienced [profession] with [specialization]
- [Role] explaining to [audience]
- Critical reviewer / devil's advocate

### 4. Output Format Specification

Always specify desired format:

```
Provide your response in this format:

## Summary
[2-3 sentence overview]

## Key Points
- [Point 1]
- [Point 2]
- [Point 3]

## Action Items
1. [Action 1]
2. [Action 2]

## Risks
| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|------------|
| [Risk] | High/Med/Low | High/Med/Low | [Strategy] |
```

## Core Techniques

### Zero-Shot Prompting

Direct instruction without examples:

```
Classify the sentiment of this product review as Positive, Negative, or Neutral:

Review: "The battery life is amazing but the camera quality is disappointing."
Sentiment:
```

**Best for:**
- Simple, well-defined tasks
- When examples aren't available
- Testing model baseline capabilities

### Few-Shot Prompting

Provide examples to guide the model:

```
Classify the sentiment of product reviews.

Review: "Best purchase I've made this year!"
Sentiment: Positive

Review: "Completely broke after one week."
Sentiment: Negative

Review: "It's okay, nothing special."
Sentiment: Neutral

Review: "The battery life is amazing but the camera quality is disappointing."
Sentiment:
```

**Best for:**
- Specific formatting requirements
- Complex classification tasks
- Teaching the model a pattern

### Chain-of-Thought (CoT)

Encourage step-by-step reasoning:

```
Q: A farmer has 10 cows. 3 die, 2 are sold, and he buys 5 more. 
   How many cows does he have?

A: Let's work through this step by step:
   - Starting amount: 10 cows
   - After 3 die: 10 - 3 = 7 cows
   - After selling 2: 7 - 2 = 5 cows
   - After buying 5: 5 + 5 = 10 cows
   - Final answer: 10 cows

Q: [Your complex question]
A: Let's work through this step by step:
```

**Why it works:**
- Forces model to show reasoning
- Reduces errors on complex problems
- Makes mistakes easier to spot
- Improves accuracy on math/logic tasks

### Self-Consistency

Generate multiple answers and take the majority:

```
Solve this problem. Provide your reasoning, then state your final answer.

[Problem]

Reasoning:
```

Run 3-5 times, select most common answer.

## Advanced Techniques

### Tree of Thoughts (ToT)

Explore multiple reasoning paths:

```
Problem: [Complex problem]

Explore 3 different approaches to solve this:

Approach 1: [Method name]
Reasoning:
- Step 1
- Step 2
- Step 3
Result: [Outcome]

Approach 2: [Method name]
Reasoning:
- Step 1
- Step 2
- Step 3
Result: [Outcome]

Approach 3: [Method name]
Reasoning:
- Step 1
- Step 2
- Step 3
Result: [Outcome]

Evaluation:
Compare the approaches and select the best solution.

Final Answer: [Best approach with reasoning]
```

### ReAct (Reasoning + Acting)

Interleave reasoning with tool use:

```
You have access to a calculator and web search.

Question: What is the population of Tokyo divided by the population of New York City?

Thought 1: I need to find the population of Tokyo.
Action 1: Search[population of Tokyo 2024]
Observation 1: Tokyo population is approximately 13.9 million.

Thought 2: Now I need the population of New York City.
Action 2: Search[population of New York City 2024]
Observation 2: NYC population is approximately 8.3 million.

Thought 3: Now I can calculate the ratio.
Action 3: Calculate[13.9 / 8.3]
Observation 3: Result is approximately 1.67

Final Answer: The population of Tokyo is approximately 1.67 times that of New York City.
```

### System Prompting

Set persistent behavior:

```
System: You are a code reviewer focused on security vulnerabilities. 
Be thorough, cite specific lines, and suggest fixes with code examples.

User: Review this Python function:
def authenticate(username, password):
    query = f"SELECT * FROM users WHERE username='{username}' AND password='{password}'"
    return db.execute(query)
```

### Meta-Prompting

Ask the model to improve your prompt:

```
I want to create a prompt that generates marketing copy for SaaS products.

Current prompt: "Write marketing copy for a SaaS product"

Please improve this prompt by:
1. Adding specific instructions for tone and style
2. Including placeholders for product details
3. Specifying the target audience
4. Requesting specific elements (headlines, benefits, CTA)

Provide the improved prompt and explain your changes.
```

## Domain-Specific Patterns

### For Code Generation

```
Language: Python 3.11
Task: Write a function that validates email addresses

Requirements:
- Use regex for validation
- Return tuple: (is_valid: bool, error_message: str)
- Handle common edge cases
- Include type hints
- Add docstring with examples
- Include 3 unit tests

Function signature:
def validate_email(email: str) -> tuple[bool, str]:
```

### For Data Analysis

```
Role: Senior Data Analyst

Dataset: Customer churn data with columns:
- customer_id, tenure, monthly_charges, total_charges
- contract_type, payment_method, churn (yes/no)

Task: Identify top 3 factors correlated with churn

Deliverables:
1. Statistical analysis approach
2. Key findings with visualizations described
3. Business recommendations
4. SQL queries used (if applicable)

Format: Executive summary suitable for non-technical stakeholders
```

### For Creative Writing

```
Genre: Science Fiction
Tone: Hopeful but realistic
Length: 500 words

Prompt: Write a scene set in 2150 where humans have successfully 
coexisted with AI for 50 years. Focus on a everyday interaction 
between a human and their AI companion.

Constraints:
- Show, don't tell
- Include sensory details
- Avoid clichés about robot uprisings
- Emphasize the mundane, not the extraordinary
```

## Common Pitfalls and Solutions

### Pitfall 1: Over-Engineering

**Problem:** 500-word prompt for simple task

**Solution:** Start simple, add complexity only if needed

### Pitfall 2: Ambiguous Instructions

**Problem:** "Make it better"

**Solution:** Define specific criteria for "better"

### Pitfall 3: Contradictory Constraints

**Problem:** "Be comprehensive but brief"

**Solution:** Prioritize constraints or provide relative weightings

### Pitfall 4: Assuming Context

**Problem:** Referencing previous conversations without context window

**Solution:** Include relevant context in each prompt

### Pitfall 5: Ignoring Token Limits

**Problem:** Prompt + context exceeds model's context window

**Solution:** Use chunking, summarization, or retrieval-augmented generation

## Optimization Strategies

### A/B Testing Prompts

Create variations and compare:

```
Version A: "Explain quantum computing to a 10-year-old"
Version B: "Explain quantum computing using only analogies a child would understand"
Version C: "You're a science teacher explaining quantum computing to 5th graders. Use the double-slit experiment as your main example."
```

Evaluate on:
- Accuracy
- Clarity
- Engagement
- Appropriate complexity

### Iterative Refinement

1. **Baseline**: Run initial prompt
2. **Analyze**: Identify weaknesses
3. **Adjust**: Modify specific elements
4. **Test**: Compare results
5. **Repeat**: Continue until satisfied

### Prompt Chaining

Break complex tasks into steps:

```
Step 1: "Summarize this research paper in 3 bullet points"
[Output]

Step 2: "Based on these bullet points, identify the key methodology"
[Output]

Step 3: "Evaluate this methodology's strengths and weaknesses"
```

## Measuring Prompt Effectiveness

### Metrics

**Task Completion**
- Did the model perform the requested task?
- Binary: Success/Failure

**Output Quality**
- Relevance to prompt
- Accuracy of information
- Appropriateness of tone
- Subjective rating (1-5)

**Efficiency**
- Tokens used
- Number of iterations needed
- Time to satisfactory result

### Evaluation Framework

```
For each prompt variation:
1. Run 10 times with different inputs
2. Score outputs on quality metrics
3. Calculate average and variance
4. Identify failure modes
5. Document edge cases
```
